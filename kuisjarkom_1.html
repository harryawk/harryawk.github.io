SQL Injection

Pendahuluan

SQL injection merupakan suatu teknik yang menyalahgunakan celah keamanan basis data dari suatu aplikasi. Teknik ini disebut SQL injection karena menyisipkan kode SQL yang sebenarnya tidak diinginkan oleh pembuat aplikasi. Kode SQL ini dapat disisipkan di form atau pada http get (url).

Skenario Pengujian dan Hasil Pengujian

Misal pada form

Sepintas tak ada yang aneh, namun orang yang telah memprogram web akan dapat mengira kode SQL yang digunakan untuk login. Seperti 

$query = "SELECT * FROM `user` where `username`='$username' and `password`='$password';"
dimana $username dan $password akan berisi string yang diisikan pada form.

Lalu bagaimana jika kita mengisinya dengan kode SQL? membuatnya menjadi mengeksekusi pertintah yang tidak sesuai? misalnya dengan mengisi username sebagai "'; -- OR 1=1" akan mengubah query menjadi SELECT * FROM `user` where `username`='' OR 1=1; -- ' and `password`='$password';

dimana yang berwarna hitam adalah kode sql yang dieksekusi, sementara teks abu-abu akan menjadi komentar dan akan diabaikan. OR 1=1 akan mengakibatkan perintah SQL menjadi selalu benar dan akan mengembalikan semua yang ada di tabel user.

Sampai saat ini SQL injection masih menjadi masalah keamanan. Sebagai contoh adalah website yang ditemukan oleh penulis ini.





Simpulan

SQL injection dapat berdampak fatal, terutama pada laman login. Oleh karena itu sebisa mungkin dicegah. Sudah banyak bahasa pemrograman yang memperkenalkan konsep prepared statement yang dapat mencegah SQL injection. Prepared statement adalah mengirim query awal dan parameter data secara terpisah lalu parameter yang dikirim tetap diolah sebagai parameter.
